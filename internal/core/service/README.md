# internal/core/service 设计说明

## 设计定位
该模块是用例编排层，负责把存储、缓存、模型和 RAG 串成业务闭环。

## 核心设计思路
1. 采用接口先行架构，先定义 repository/cache/model/queue/rag 接口，再落地具体实现。
2. 查询和流式走相同领域流程：写入用户消息、组装上下文、模型生成、持久化回复。
3. 通过超时、回退、幂等与摘要缓存控制主链路稳定性。

## 边界与依赖
边界在于服务层只关心用例，不关心 Gin/RabbitMQ 细节；依赖抽象接口而非具体实现。

## 演进策略
后续可拆分为 application/domain service，并加入策略引擎。
