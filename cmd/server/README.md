# cmd/server 设计说明

## 设计定位
该模块是系统启动编排层，负责把配置、基础设施、业务服务和 HTTP 路由按依赖顺序组装起来。

## 核心设计思路
1. 采用组合根模式，把依赖注入集中在 main，避免业务层反向依赖基础设施。
2. 将可替换能力通过接口注入，保证后续替换不影响路由层。
3. 启动与关闭走统一生命周期管理，保证服务可观测并可优雅退出。

## 边界与依赖
边界在于只做装配不做业务决策；依赖 internal/config、repo、queue、model、transport。

## 演进策略
后续可引入更标准的 DI 容器或模块化启动器，但仍保持 main 只承担编排职责。
